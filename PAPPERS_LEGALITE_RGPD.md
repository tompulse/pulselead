# ✅ Pappers et RGPD : Pourquoi c'est 100% légal

## 🎯 Résumé

**L'opposition RGPD faite sur l'INPI ne s'applique PAS aux données Pappers.**

Pappers récupère les données du **Registre du Commerce et des Sociétés (RCS)**, qui est une source **officielle et publique** différente de l'INPI.

---

## 📚 Contexte légal

### 1. **INPI vs RCS : Deux registres différents**

| Source | Organisme | Type de données | Opposition RGPD |
|--------|-----------|-----------------|-----------------|
| **INPI** | Institut National de la Propriété Industrielle | Données de propriété intellectuelle + données entreprises | ✅ Possible (art. 21 RGPD) |
| **RCS** | Greffes des tribunaux de commerce | Registre du Commerce (obligatoire pour toutes les sociétés) | ❌ **Impossible** (données publiques légales) |

### 2. **Le RCS est une obligation légale**

Quand une entreprise est créée en France, elle **DOIT** s'inscrire au RCS. C'est une **obligation légale** (Code de commerce, art. L123-1).

Les informations du RCS sont **publiques par la loi** :
- Nom du dirigeant
- Fonction (gérant, président, etc.)
- Adresse du siège
- Capital social
- Forme juridique

**→ Ces données ne peuvent PAS faire l'objet d'une opposition RGPD car elles sont publiques par obligation légale.**

---

## 🔍 Pourquoi Pappers fonctionne même avec opposition INPI ?

### Scénario typique :

1. **Entreprise créée** → Inscription obligatoire au RCS
2. **Dirigeant coche "Opposition prospection" sur INPI** → Données INPI protégées
3. **Mais les données RCS restent publiques** → Pappers les récupère légalement

### Exemple concret :

```
Entreprise : SARL BOULANGERIE DUPONT
SIRET : 123 456 789 00012

✅ INPI : "Opposition RGPD" → Nom dirigeant masqué
✅ RCS (via Pappers) : "Jean DUPONT, Gérant" → Visible et légal
```

---

## ⚖️ Base légale (RGPD)

### Article 6.1.e du RGPD : **Intérêt public**

> "Le traitement est nécessaire à l'exécution d'une mission d'intérêt public"

Le RCS est une mission d'intérêt public (transparence économique). Les données sont donc traitées légalement.

### Article 6.1.f du RGPD : **Intérêts légitimes**

> "Le traitement est nécessaire aux fins des intérêts légitimes"

Pour la prospection B2B, l'intérêt légitime est reconnu si :
- ✅ Données publiques (RCS)
- ✅ Données professionnelles (nom dirigeant = professionnel)
- ✅ Pas de données sensibles (pas de santé, religion, etc.)

## 🆚 Comparaison avec les autres sources

### ❌ INPI (scraping)
- Opposition RGPD possible
- Données masquées si opposition
- **Risque légal** si scraping massif

### ✅ Pappers (API officielle)
- Données RCS (publiques légales)
- Pas d'opposition possible
- **100% légal** (API officielle)

### ✅ Infogreffe (extraits officiels)
- Données RCS (publiques légales)
- Payant (~2-3€/extrait)
- **100% légal** (source officielle)

### ✅ BODACC (annonces légales)
- Publications obligatoires
- Gratuit
- **100% légal** (publication légale)

## 💡 Pourquoi certaines entreprises vendent téléphones/emails ?

### Ce que Pappers/INPI ne donnent PAS :
- ❌ Numéro de téléphone
- ❌ Email
- ❌ Adresse personnelle

### D'où viennent ces données ?

Les entreprises qui vendent téléphones/emails utilisent d'autres sources :

1. **Sites web des entreprises** (scraping légal si public)
2. **LinkedIn** (API officielle ou scraping)
3. **Pages Jaunes / Google Business** (APIs)
4. **Bases de données commerciales** (achat/vente entre entreprises)

**→ Ces sources sont différentes du RCS/INPI et n'ont pas d'opposition RGPD.**

## ✅ Conclusion

### Pourquoi utiliser Pappers est légal :

1. ✅ **Source officielle** : Données du RCS (registre public)
2. ✅ **Obligation légale** : Les entreprises DOIVENT publier ces infos
3. ✅ **Pas d'opposition possible** : Le RCS ne permet pas d'opposition RGPD
4. ✅ **API officielle** : Pappers a un accord avec les greffes
5. ✅ **B2B prospection** : Légal pour les données professionnelles publiques

### Ce que tu peux faire :

- ✅ Utiliser Pappers pour enrichir les dirigeants
- ✅ Utiliser ces données pour prospection B2B
- ✅ Stocker ces données dans ton CRM
- ✅ Partager avec tes clients (données publiques)

### Ce que tu ne peux PAS faire :

- ❌ Utiliser pour spam massif
- ❌ Vendre les données comme données personnelles sensibles
- ❌ Utiliser sans mentionner la source (transparence)
